# Release Authority Infrastructure

The release authority system creates release-candidate evidence. It does not bypass governed production gates.

## Commands

```bash
npm run release:sign
npm run release:verify
npm run govern:release
```

## Outputs

`npm run release:sign` writes:

- `release/release-authority/release-authority.json`
- `release/release-authority/signed-release-manifest.json`
- `release/release-authority/release-lineage-history.json`
- `release/release-authority/operator-approval-log.json`
- `release/release-authority/release-certificate-registry.json`
- `release/evidence/release-authority.json`
- `release/evidence/release-authority.md`

## Signers

The implementation verifies externally supplied signatures. It does not create software fallback signatures.

Supported signer inputs:

```bash
export MAATAA_RELEASE_TPM2_PUBLIC_KEY_PEM=/secure/path/tpm2-release-public.pem
export MAATAA_RELEASE_TPM2_SIGNATURE_B64='base64-signature'
export MAATAA_RELEASE_TPM2_CERTIFICATE_PEM=/secure/path/tpm2-cert.pem

export MAATAA_RELEASE_HSM_PUBLIC_KEY_PEM=/secure/path/hsm-release-public.pem
export MAATAA_RELEASE_HSM_SIGNATURE_B64='base64-signature'
export MAATAA_RELEASE_HSM_CERTIFICATE_PEM=/secure/path/hsm-cert.pem
```

The signature must verify over the exact JSON release manifest payload generated by the release authority script.

## Operator Approval Chain

Operator approvals must be real signed records. Configure:

```bash
export MAATAA_RELEASE_APPROVALS_JSON=/secure/path/release-approvals.json
export MAATAA_RELEASE_QUORUM=2
```

Each approval must include:

```json
{
  "operator_id": "brahmini",
  "role": "Admin",
  "approved_at": "2026-05-26T00:00:00.000Z",
  "public_key_pem": "-----BEGIN PUBLIC KEY-----...",
  "signature_b64": "base64-signature"
}
```

The signed approval payload is:

```json
{
  "schema": "maataa.release.operator-approval.payload.v1",
  "release_manifest_hash": "manifest hash",
  "operator_id": "brahmini",
  "role": "Admin",
  "approved_at": "2026-05-26T00:00:00.000Z"
}
```

## Gate Behavior

- No unsigned production releases.
- No fake signatures.
- No fake quorum approvals.
- Missing signer or missing quorum writes BLOCKED evidence.
- A verified signer plus verified quorum can produce `GOVERNED_RELEASE_CANDIDATE`.
- Production GO still requires the existing governed production gates.
